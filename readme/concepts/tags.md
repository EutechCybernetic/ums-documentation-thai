# แท็ก

แท็กเป็นคุณสมบัติที่มีประโยชน์ในการจัดระเบียบและจัดแคตตาล็อกข้อมูลการใช้งานสาธารณูปโภคในทางที่แตกต่างกัน คุณสามารถใช้แท็กหลายๆ อันในมิเตอร์, มิเตอร์เสมือนหรือกลุ่มมิเตอร์ คุณจึงสามารถวิเคราะห์ข้อมูลการใช้งานตามแท็ก มิเตอร์ทั้งหมดที่มีแท็กเดียวกันจะมีข้อมูลการใช้งานรวมกัน เช่น ถ้ามีผู้เช่าหลายคนในอาคารและพวกเขามีมิเตอร์หลายๆ ตัว คุณสามารถใช้แท็กในมิเตอร์หรือกลุ่มมิเตอร์ต่างๆ ด้วยชื่อผู้เช่า แล้ววิเคราะห์ข้อมูลการใช้งานตามชื่อผู้เช่า

เพื่อให้มีโครงสร้างและรับรองว่าข้อมูลสะอาด คุณต้องกำหนดแท็กของคุณล่วงหน้า

คุณเริ่มต้นด้วยการกำหนด 'ประเภทแท็ก' - เช่น 'ผู้เช่า' คุณสามารถกำหนดรายการหลายๆ อันภายใต้แท็ก 'ผู้เช่า' คุณสามารถเพิ่มแท็กสำหรับแต่ละผู้เช่า - 'ACME', 'Blueberry Electronics', 'Verper Stationary' ฯลฯ..

สิ่งที่ทำให้แท็กน่าสนใจมากขึ้นคือ พวกเขาสามารถปฏิบัติตามโครงสร้างต้นไม้แบบลำดับชั้น

#### ต้นไม้แท็ก

คุณสามารถกำหนดแท็กที่มีแท็กแม่และแท็กลูก คุณสามารถสร้างแท็กและแท็กย่อยและแท็กย่อยย่อยและยังคงทำต่อไป ข้อดีของสิ่งนี้คือ เมื่อคุณต้องการวิเคราะห์ข้อมูลการใช้งานสำหรับแท็ก มันจะรวมแท็กย่อยทั้งหมด (หรือแท็กลูก) ด้วย เช่น ถ้าผู้เช่า 'Sunbucks Coffee' มีร้าน 2 แห่งในอาคารพาณิชย์ คุณสามารถกำหนดโครงสร้างแท็กสำหรับผู้เช่าดังนี้:

* ผู้เช่า
  * อาหารและเครื่องดื่ม
    * Sunbucks Coffee
      * ชั้นล่าง
      * ชั้น 3
    * Don Pepe

คุณสามารถแท็กมิเตอร์เป็น `ผู้เช่า/อาหารและเครื่องดื่ม/Sunbucks Coffee/ชั้นล่าง` หรือ `ผู้เช่า/อาหารและเครื่องดื่ม/Sunbucks Coffee/ชั้น 3`.

คุณสามารถวิเคราะห์ข้อมูลสำหรับแต่ละมิเตอร์ หรือถ้าคุณเลือกวิเคราะห์ข้อมูลสำหรับ `ผู้เช่า/อาหารและเครื่องดื่ม/Sunbucks Coffee`, ข้อมูลจาก `ชั้นล่าง` และ `ชั้น 3` จะถูกรวมอย่างอัตโนมัติ

ถ้าคุณวิเคราะห์ข้อมูลสำหรับแท็ก `ผู้เช่า/อาหารและเครื่องดื่ม` มันจะรวมข้อมูลการใช้งานจากมิเตอร์ที่มีแท็ก `Don Pepe` ด้วย

การมีโครงสร้างต้นไม้ช่วยให้คุณแยกและวิเคราะห์ข้อมูลแบบเดี่ยวหรือเป็นกลุ่ม

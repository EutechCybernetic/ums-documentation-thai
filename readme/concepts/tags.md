# แท็ก

แท็กเป็นคุณสมบัติที่มีประโยชน์ในการจัดระเบียบและจัดแคตตาล็อกข้อมูลการใช้งานสาธารณูปโภคในทางที่แตกต่างกัน คุณสามารถใช้แท็กหลายๆ ตัวกับเมตร, เมตรเสมือน หรือกลุ่มเมตร คุณสามารถวิเคราะห์ข้อมูลการใช้งานตามแท็ก ทุกเมตรที่มีแท็กเดียวกันจะมีข้อมูลการใช้งานที่ถูกรวมกัน เช่น ถ้ามีผู้เช่าหลายคนในอาคารและแต่ละคนมีเมตรหลายตัว คุณสามารถใช้แท็กเมตรหรือกลุ่มเมตรที่แตกต่างกันด้วยชื่อผู้เช่า แล้ววิเคราะห์ข้อมูลการใช้งานตามชื่อผู้เช่า

เพื่อให้มีโครงสร้างและข้อมูลที่สะอาด คุณต้องกำหนดแท็กของคุณล่วงหน้า

คุณเริ่มต้นด้วยการกำหนด 'ประเภทแท็ก' - เช่น 'ผู้เช่า' คุณสามารถกำหนดรายการหลายๆ รายการใต้แท็ก 'ผู้เช่า' คุณสามารถเพิ่มแท็กสำหรับแต่ละผู้เช่า - 'ACME', 'Blueberry Electronics', 'Verper Stationary' ฯลฯ..

สิ่งที่ทำให้แท็กน่าสนใจมากขึ้นคือ พวกมันสามารถปฏิบัติตามโครงสร้างต้นไม้แบบลำดับชั้น

#### ต้นไม้แท็ก

คุณสามารถกำหนดแท็กที่มีแท็กแม่และแท็กลูก คุณสามารถสร้างแท็กและแท็กย่อยและแท็กย่อยย่อยและยังคงทำต่อไป ข้อดีของสิ่งนี้คือ เมื่อคุณต้องการวิเคราะห์ข้อมูลการใช้งานสำหรับแท็ก มันจะรวมทุกแท็กย่อย (หรือแท็กลูก) ด้วย เช่น ถ้าผู้เช่า 'Sunbucks Coffee' มีร้าน 2 แห่งในอาคารพาณิชย์ คุณสามารถกำหนดโครงสร้างแท็กสำหรับผู้เช่าดังนี้:

* ผู้เช่า
  * อาหารและเครื่องดื่ม
    * Sunbucks Coffee
      * ชั้นล่าง
      * ชั้น 3
    * Don Pepe

คุณสามารถแท็กเมตรเป็น `ผู้เช่า/อาหารและเครื่องดื่ม/Sunbucks Coffee/ชั้นล่าง` หรือ `ผู้เช่า/อาหารและเครื่องดื่ม/Sunbucks Coffee/ชั้น 3`.

คุณสามารถวิเคราะห์ข้อมูลสำหรับแต่ละเมตร หรือถ้าคุณเลือกวิเคราะห์ข้อมูลสำหรับ `ผู้เช่า/อาหารและเครื่องดื่ม/Sunbucks Coffee` ข้อมูลจาก `ชั้นล่าง` และ `ชั้น 3` จะถูกรวมอย่างอัตโนมัติ

ถ้าคุณวิเคราะห์ข้อมูลสำหรับแท็ก `ผู้เช่า/อาหารและเครื่องดื่ม` มันจะรวมข้อมูลการใช้งานจากเมตรที่มีแท็ก `Don Pepe` ด้วย

การมีโครงสร้างแบบต้นไม้ช่วยให้คุณแยกและวิเคราะห์ข้อมูลแบบเดี่ยวหรือเป็นกลุ่ม

# มิเตอร์เสมือนจริง

มิเตอร์เสมือนจริง - หรือ 'มิเตอร์ซอฟต์แวร์' - เป็นมิเตอร์ที่ถูกกำหนดในชั้นซอฟต์แวร์และมีค่าที่ได้มาจากมิเตอร์ทางกายภาพหนึ่งหรือหลายตัว โดยทั่วไป, อาคารอาจจะไม่มีมิเตอร์สำหรับทุกๆ ระบบย่อยที่ต้องการวัด - แต่คุณสามารถคำนวณค่าพลังงานสำหรับระบบย่อยนั้นๆ โดยการสืบทอดข้อมูลจากมิเตอร์ทางกายภาพอื่นๆ 

ตัวอย่างเช่น, ถ้าชั้นมีมิเตอร์สำหรับมัน (เราเรียกมันว่า F1), และแต่ละหน่วยภายในชั้นมีมิเตอร์ของตัวเอง (เราเรียกมันว่า U1 และ U2), แล้วเราสามารถวัดพลังงานทั้งหมดที่เข้าไปในชั้น และพลังงานที่เข้าไปในแต่ละหน่วย

แต่พลังงานที่ถูกใช้โดยพื้นที่ส่วนกลางบนชั้นนั้นล่ะ? ไม่มีมิเตอร์แยกเพื่อวัดสิ่งนั้น แต่เราสามารถ _คำนวณ_ ได้โดยการนำค่าการใช้พลังงานจาก F1 แล้วลบด้วยค่าการใช้พลังงานจาก U1 และ U2 พลังงานที่เหลือคือพลังงานของพื้นที่ส่วนกลางในชั้น

ในกรณีนี้, คุณสามารถกำหนดมิเตอร์เสมือนจริงที่เรียกว่า 'CA1' เพื่อแทนการใช้พลังงานของพื้นที่ส่วนกลาง และคุณสามารถกำหนดค่าให้มันตามสูตร `F1 - U1 - U2`

เมื่อกำหนดค่ามิเตอร์เสมือนจริง, คุณต้องระบุสถานที่ให้บริการ, ชื่อ และแท็ก นอกจากนี้คุณยังต้องระบุว่ามิเตอร์เสมือนจริงนี้สืบทอดข้อมูลของมันมาจากมิเตอร์ใดและปัจจัยการขยายสำหรับแต่ละมิเตอร์นั้น

ใช้ตัวอย่างด้านบน - `F1 - U1 - U2`, คุณจะต้องเพิ่ม 3 มิเตอร์และปัจจัยการขยายที่สอดคล้องดังนี้:

| มิเตอร์ | ปัจจัยการขยาย |
| ----- | -------------- |
| F1    | 1              |
| U1    | -1             |
| U2    | -1             |

#### หน่วยสำหรับมิเตอร์เสมือนจริง

มิเตอร์เสมือนจริงสามารถคำนวณได้จากมิเตอร์ที่มีหน่วยต่างกัน การใช้พลังงานทั้งหมดสำหรับมิเตอร์เสมือนจริงจะถูกเก็บในหน่วยพื้นฐานของประเภทสาธารณูปโภค ดังนั้นถ้าคุณได้กำหนดมิเตอร์น้ำเสมือนจริงและประเภทสาธารณูปโภค `น้ำ` มี 'แกลลอนอิมพีเรียล' เป็นหน่วยพื้นฐาน (เช่น, ปัจจัยการแปลงของหน่วยเป็น 1), แล้วข้อมูลการใช้พลังงานของมิเตอร์เสมือนจริงทั้งหมดจะถูกคำนวณในแกลลอนอิมพีเรียล ไม่ว่าหน่วยของมิเตอร์ที่มิเตอร์เสมือนจริงได้รับมาจากจะเป็นอย่างไร

#### การคำนวณการใช้พลังงาน

การใช้พลังงานสำหรับมิเตอร์เสมือนจริงจะถูกคำนวณอัตโนมัติเมื่อมีการบันทึกข้อมูลการใช้พลังงานในมิเตอร์ทางกายภาพใดๆ
